    Preguntas

    Pregunta 1: ¿Cuál es la diferencia entre Internet y la World Wide Web?

    R1: El internet es una red global de computadoras interconectadas. En el se usa un servicio llamado World Wide Web. Dicho servicio es un sistema de documentación e hipervínculos funcionando por el internet.

    Pregunta 2: ¿Cuáles son las partes de un URL?

    • Protocolo
    • Nombre de dominio
    • Puerto
    • Ruta
    • Query String
    • Parámetros

    Pregunta 3: ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    • Get: Se utiliza para solicitar información de un recurso específico.
    • Head: Similar a GET, pero solo devuelve los encabezados de la respuesta.
    • Post: Se utiliza para enviar datos al servidor y crear un nuevo recurso.
    • Put: Se utiliza para actualizar un recurso existente o crear uno nuevo si no existe.
    • Patch: Se utiliza para realizar actualizaciones parciales en un recurso existente.
    • Delete: Se utiliza para eliminar un recurso específico.

    Pregunta 4: ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    El método post. Esto es debido a que está diseñado para enviar y crear datos. Por lo que usa un cifrado en conjunto con HTTP.

    Pregunta 5: ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    El método get. Esto es debido a que se utiliza para solicitar información de un recurso específico, como una página web.

    Pregunta 6: Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    El código 200 significa que la petición de la búsqueda fue exitosa. No ocurre ningún error, se despliega el recurso en el cuerpo del mensaje.

    Pregunta 7: ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

      Un atributo HTML5 desaprobado significa que, siendo soportado por navegadores compatibles con código antiguo, su uso se desaconseja. Algunos elementos desaprobados son:
    • acronym — Define acronimos
    • basefont — Define tipografía base
    • center — Centra texto
    • coords — Define coordenadas para definir formas
      • Recuperado de: (Etiquetas y atributos obsoletos de HTML 4.01, n.d.) Las alternativas que se han desarrollado son el uso de CSS o diferentes comandos con la misma funcionalidad.

    Pregunta 8: ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    Depende del origen del error. El error 404 explica que el recurso no existe. Por lo que el error puede ser tanto del usuario por una URL incorrecta como puede ser problema del sitio. Pero es responsabilidad del desarrollador verificar la funcionalidad del sitio.

    Pregunta 9: ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    Sí, esto es porque el error 500 presenta un error de servidores internos. Generalmente significa que la aplicación crasheó. Es responsabilidad del desarrollador.

    Pregunta 10: ¿Qué componentes de estructura y estilo tiene una tabla?

    Componentes de estructura:

    • table — Define inicio y fin de la tabla
    • tr — Contiene las celdas de una fila
    • th — Maneja los encabezados
    • td — Celdas comunes donde se insertan los datos
    • caption — Título de la tabla

    Componentes de estilo (mediante CSS):

    • width — Ancho de la tabla o celda
    • border-collapse: collapse — Elimina espacios entre bordes
    • background-color — Color de fondo
    • color — Color del texto
    • text-align — Alineación del texto

    Pregunta 12: Sobre el ciclo de vida y desarrollo de los sistemas de información

    ¿Cuál es el ciclo de vida de los sistemas de información?

    • Concepción: Surge la necesidad
    • Desarrollo: Análisis, diseño y construcción
    • Implementación: Producción y capacitación
    • Operación: Sistema en uso
    • Mantenimiento: Corrección de errores y mejoras
    • Retiro: Sistema descontinuado

    ¿Cuál es el ciclo de desarrollo de sistemas de información?

    • Planeación: Definición de alcance y recursos
    • Análisis: Requerimientos de usuario
    • Diseño: Arquitectura, base de datos e interfaces
    • Desarrollo: Código y construcción del sistema
    • Pruebas: Verificación de funcionamiento
    • Implementación: Entrega del sistema
    • Mantenimiento: Soporte y mejoras
