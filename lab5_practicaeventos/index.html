Preguntas

    Porque es una buena practica usar JS para checar que sean validos los inputs de las formas antes de enviarlos al servidor?
  • Porque evita procesos innecesarios en el servidor. Si el usuario ingresa o no algo incorrecto, JS lo detecta y hace una experiencia optima.

    • Como puedes saltarte validaciones en JS?
  • Desactivando JS en el navegador.
  • Devtools para modificar HTML
  • Peticion a traves de POSTMAN saltandose al navegador

    • Si se puede saltar, porque es buena practica?
  • Porque validaciones en JS y servidor son diferentes propositos. Mejora la experiencia del usuario JS y el servidor mejora la seguridad. En realidad necesitamos las dos.
