Preguntas Reflexion

1. ¿Qué ventajas ofrecen las variables CSS sobre los valores hardcodeados?

Las variables en css permiten reutilizar codigo, eficiencia, mantenimiento y optimizabilidad. Manejan consistencia y falicitan uso.

2. ¿En qué casos conviene declarar variables CSS fuera del selector :root?

Conviene cuando se necestan variables especificas. Manejar dentro de un modulo y no afectar a las secciones globales.

3. ¿Cómo se pueden utilizar las variables CSS para implementar un sistema de temas?

Al usar root, manejamos un sistema de temas, esto para manejar una constancia en el diseno, y asi poder cambiar el tema de manera facil.

4. ¿Qué sucede cuando intentas usar una variable CSS que no ha sido declarada?

La declaracion del uso de la variable resulta en un valor inicial. Prederteminado, esto no genero errores de sintaxis.

5. ¿Cuál es la diferencia entre los enfoques mobile-first y desktop-first?

Con Mobile-first, Se disena para pantallas pequenas y se modifica para pixeles mas grandes. Desktop-First, mismo concepto pero al reves. Disenas hacia abajo. La Desktop permite complejidad pero afecta rendimiento movil.

6. ¿Qué breakpoints son los más comunes y por qué?

Los breakpoints más comunes son: 320px, 480px, 768px, 1024px y 1200px. Estos se utilizan porque representan tamaños de pantalla comunes en dispositivos móviles y de escritorio.

7. ¿Cómo afecta el viewport meta tag al diseño responsivo?

Es fundamental porque es el componente que dice como ajustar las dimensiones en los diferentes dispositivos.

8. ¿Qué técnicas además de media queries existen para crear diseños responsivos?

Uso de unidades relativas, diseno de Flexbox y Grid layout.

9. ¿Cómo puedes identificar qué regla CSS está siendo aplicada cuando hay múltiples selectores?

Se identifica a traves de el inspector de elementos en el navegador.

10. ¿Que informacion proporciona la pestana Computed en DevTools?

La pestana Computed en Devtools proporciona la informacion sobre los valores de CSS finales dentro del codigo.

11. ¿Cómo puedes simular diferentes dispositivos sin tener acceso físico a ellos?

Esto se simula a traves de las herramientas del inspector del navegador. Al seleccionar los dispositivos en la pestana de emuladores, podemos seleccionar los dispositivos en diferentes, y asi simular el comportamiento de la pagina en diferentes pantallas.

12. ¿En qué situaciones es preferible usar Bootstrap sobre Tailwind CSS y viceversa?

Es preferible cuando no necesitas una personalizacion de estilos. Se usa Bootstrap para manejar generalidad dentro de estilos.

13. ¿Qué significa que Tailwind sea "utility-first"?

Utility-first significa que se construyen clases de utilidad para cada propiedad dentro de los estilos. Esto maneja persinalizacion e individualismo.

14. ¿Cómo afecta el uso de un framework CSS al tamaño final del bundle?

El uso de un framework CSS tiene un impacto en el tamano de l paquete de archivos. Pudiendo aumentarlo si no se gestiona correctamente mediante tecnicas de optimizacion modernas.

15. ¿Es posible combinar múltiples frameworks CSS en un mismo proyecto?

Si, es posible combinar multiples frameworks CSS en un mismo proyecto, pero esto puede generar conflictos de estilos y aumentar la complejidad del mantenimiento.
